# Инструкция по нагрузочному тестированию проекта "Чилинкин"

## Подготовка к тестированию

### 1. Установка зависимостей
```bash
# Активируем виртуальное окружение
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\Activate  # Windows

# Устанавливаем зависимости для тестов
pip install -r requirements_test.txt
```

### 2. Настройка тестовой среды
```bash
# Применяем миграции
python manage.py migrate

# Создаем тестовые данные (опционально)
python manage.py shell << EOF
from tickets.models import Role, ServiceType, TicketStatus
from django.contrib.auth import get_user_model

User = get_user_model()

# Создаем роли если их нет
Role.objects.get_or_create(name=Role.CLIENT)
Role.objects.get_or_create(name=Role.WORKER)
Role.objects.get_or_create(name=Role.ADMIN)
Role.objects.get_or_create(name=Role.DISPATCHER)

print("Тестовые данные созданы")
EOF
```

## Запуск нагрузочных тестов

### Запуск всех тестов
```bash
python run_load_tests.py all
```

### Запуск отдельных категорий тестов
```bash
# Только тесты входа в систему
python run_load_tests.py login

# Только тесты базы данных
python run_load_tests.py database

# Только тесты операций с заявками
python run_load_tests.py operations
```

## Ожидаемые результаты для вашего ноутбука

### Тесты входа в систему
- **10 пользователей:** < 300 мс среднее время, 100% успех
- **25 пользователей:** < 600 мс среднее время, >95% успех
- **50 пользователей:** < 1200 мс среднее время, >85% успех

### Тесты базы данных (SQLite)
- **100 записей:** < 10 мс время чтения
- **1000 записей:** < 50 мс время чтения
- **10000 записей:** < 200 мс время чтения
- **Создание записи:** < 20 мс

### Тесты операций с заявками
- **5 пользователей:** > 60 операций/минуту
- **15 пользователей:** > 150 операций/минуту
- **Среднее время операции:** < 500 мс

## Оптимизация производительности

### Если результаты неудовлетворительные:

1. **Оптимизация базы данных:**
   ```python
   # Использовать select_related для связанных объектов
   tickets = Ticket.objects.select_related('user', 'service_type')
   
   # Использовать prefetch_related для ManyToMany
   tickets = Ticket.objects.prefetch_related('status_history')
   ```

2. **Кэширование:**
   ```python
   # Добавить в settings.py
   CACHES = {
       'default': {
           'BACKEND': 'django.core.cache.backends.locmem.LocMemCache',
           'LOCATION': 'unique-snowflake',
       }
   }
   
   # Кэшировать частые запросы
   from django.core.cache import cache
   def get_user_tickets(user_id):
       cache_key = f'user_tickets_{user_id}'
       tickets = cache.get(cache_key)
       if not tickets:
           tickets = Ticket.objects.filter(user_id=user_id)
           cache.set(cache_key, tickets, 300)  # 5 минут
       return tickets
   ```

3. **Оптимизация статических файлов:**
   ```python
   # В settings.py
   STATICFILES_STORAGE = 'django.contrib.staticfiles.storage.StaticFilesStorage'
   
   # Использовать WhiteNoise для продакшена
   pip install whitenoise
   ```

## Частые проблемы

### 1. Ошибка импорта psutil
**Решение:** `pip install psutil==5.9.8`

### 2. Медленные тесты на SQLite
**Причина:** SQLite медленнее при параллельных операциях
**Решение:** Использовать PostgreSQL для продакшена

### 3. Высокое использование CPU
**Причина:** Многопоточные тесты нагружают процессор
**Решение:** Уменьшить количество потоков или использовать более мощное оборудование

### 4. Ошибки аутентификации
**Причина:** CSRF токены в тестах
**Решение:** Тесты используют Client() который обрабатывает CSRF автоматически
